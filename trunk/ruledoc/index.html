Ruledoc - GRAF Rule Documentation

Introduction

GRAF is based on the hypothesis that UML models can be implemented or defined using graph transformation rules ('rules'). The GRAF application attempts to demonstrate this by interpreting UML models as graphs, and the Actions that define their behavior as rules, or a set of rules. The models themselves are implemented using the widely-used Eclipse platform's Modeling Framework. The rules are based on the English descriptions of Actions in the UML specification. This Index contains both general comments on the GRAF implementation of Actions, and explanations of the specific rules that have been implemented.

The GRAF implementation of UML attempts to be as faithful to the definition as possible. However, when the definition is unclear assumptions must be made and it can be necessary to deviate from the definition. Pages for Actions list assumptions and deviations from the definition. The links below cover general assumptions and implementation decisions.

Note that rules are implemented when required by a particular model, so that a particular Action may not be completely implemented within GRAF. That is, rules will be written for the variant of an Action that is required by the model.

Rule Generation From GRAF

GRAF can be configured to generate DOT and image files of the various rules it uses when it starts up. At the moment, this requires opening the Properties file in an editor and changing the boolean value 'PrintRules' to 'true'. To convert the generated .dot files to images, GRAF uses the 'dot' command-line tool from the the Gaphviz suite. If you do not have this installed, the conversion to image format will not work.

Interpretation

In the images of the rules,
  • Names prefixed with a ? are variables, that is, they can match any name.
  • Names prefixed with a # are allocated unique names by GRAF.
  • Red edges and nodes are Negative Application Conditions ('NACs'). If they occur in the graph the rule will not be applied.
  • Green edges are added to the graph when the rule is applied.
  • Blue edges are deleted when the rule is applied.
