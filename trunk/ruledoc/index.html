Ruledoc - GRAF Rule Documentation

Introduction

GRAF is based on the hypothesis that UML models can be implemented or defined using graph transformation rules ('rules'). The GRAF application attempts to demonstrate this by interpreting UML models as graphs, and the Actions that define their behavior as rules. The models themselves are implemented using the widely-used Eclipse platform's Modeling Framework. The rules are based on the English descriptions of Actions in the UML specification. This Index attempts to explain the implementation of the rules based on that specification and other features of GRAF's environment, such as the rule sequence document.

Related Documents

  • The UML Superstructure document ('US') published by OMG defines the UML Actions that will be referred to in this index.
  • Within GRAF, the Rule Sequence File defines the order in which the rules are executed when a particular action is chosen.

Rule Generation From GRAF

GRAF can be configured to generate DOT files of the various rules it uses when it starts up. At the moment, this requires opening the PluginModel.java file in an editor, and uncommenting the code at lines 72- . This code is commented out by default as running the generation method slows GRAF down at startup, and generally the existing rule files are up to date anyway. To generate new images from these files, the 'dot' program from the Gaphviz suite is required.

Interpretation

In the images of the rules,
  • Names prefixed with a ? are variables, that is, they can match any name.
  • Names prefixed with a # are allocated unique names by GRAF.
  • Red edges and nodes are Negative Application Conditions ('NACs'). If they occur in the graph the rule will not be applied.
  • Green edges are added to the graph when the rule is applied.
  • Red edges are deleted when the rule is applied.
