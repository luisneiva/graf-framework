Rationale

This set of rules is included to create links between the calling object and a second object. For a full discussion of the implementation, assumptions and deviations, see the CreateLinkAction page.

Note that the use of the directional arcs between the objects to represent the associations is a GRAF-specific thing, and not related to the Eclipse UML file. The arcs represent 'instances' of the nodes defined in the UML file, and are created in the initial graph based on the link instances specified in the .modeltest file.
