Rationale and Comments

'ReadStructuralFeatureAction is a structural feature action that retrieves the values of a structural feature.' The action has an output pin onto which the result is put.

This rule assumes that a readSelfAction has already been executed. It is applied as part of an exclusive dependency (if-condition), and applies where the calling instance was also the caller of the readSelfAction. It does not directly link the feature to the output pin, but uses an arc of the same name to link the calling node and the feature.
